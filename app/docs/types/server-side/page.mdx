# Server-Side Types

Each server adapter exports its own config and factory types from its subpath.

## Commonly used imports

```ts
import type { ElysiaLoggerConfig, ElysiaContext } from "blyp/elysia";
import type { HonoLoggerConfig, HonoLoggerVariables } from "blyp/hono";
import type { ExpressLoggerConfig, ExpressLoggerMiddleware } from "blyp/express";
import type { FastifyLoggerConfig, FastifyLoggerPlugin } from "blyp/fastify";
import type { NestLoggerConfig, NestAdapterType } from "blyp/nestjs";
import type { NextJsLoggerFactory, NextJsLoggerConfig } from "blyp/nextjs";
import type { TanStackStartLoggerFactory } from "blyp/tanstack-start";
import type { SvelteKitLoggerFactory } from "blyp/sveltekit";
import type { WorkersLoggerConfig, WorkersRequestLogger } from "blyp/workers";
```

## `HttpRequestLog`

Most server adapters also export `HttpRequestLog`, which includes:

- `type: "http_request" | "http_error"`
- `method`
- `url`
- `statusCode`
- `responseTime`
- client metadata such as IP, user agent, hostname, and browser/OS hints
- optional error metadata like `error`, `stack`, `code`, `why`, `fix`, `link`, and `details`

## Where typing helps most

- `customProps(ctx)` for request-scoped metadata
- `autoLogging.ignore(ctx)` when you want context-aware suppression
- `clientLogging.validate(ctx, payload)` and `clientLogging.enrich(ctx, payload)`
- wrapped handler helpers such as `NextJsLoggerFactory`, `TanStackStartLoggerFactory`, and `SvelteKitLoggerFactory`
